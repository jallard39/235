Meeting Project Requirements

A. Functional

  1. I chose an external API, provided proof of concept by due date, and was successfully able to obtain an API key.
  2. The last term searched by the user is stored in the browser local storage. The previously entered values for the tags, loopable checkbox, and sort by drop down are also stored. On load, the last searched term and search and sort preferences will populate in their respective fields, but a fetch request will not be sent until the user hits the search button.
  3. There are currently 5 controls that the user can use to filter and display results. These are:
    • A search term field that the user types into
    • Up to five boxes for custom tags that the user can type into
    • A "loopable" checkbox
    • "Next" and "Previous" buttons that allow the user view multiple pages of results
    • A "Sort By" dropdown that the user can use to order results according to the parameter they want
  4. The app does not throw any JavaScript errors or exceptions.

B. Design & Interaction

  • Graphic design is pleasing (I hope). It doesn't resemble the Giphy Homework, at least.
  • Widgets are labeled and follow conventions. Users should be able to figure out how to navigate the app without needed external guidance.
  • Errors are handled gracefully. If users run a search without entering any terms, the API returns results based on its own scoring algorthim. It essentially displays the sounds it would show on the homepage. If the user leaves one of the tag boxes blank instead of removing it, it will simply be excluded from the get request. The previous and next buttons are disabled on the first and last pages, respectively, and all search buttons are disabled while fetching a request, to prevent multiple get requests.
  • The text above the results box displays the status of the program. Users will also be able to tell that the program is loading because the buttons are disabled. I would've like to add some kind of spinner, but I ran out of time.
  • I haven't tested every edge case, but the page is somewhat responsive and can gracefully handle browsers of various widths. The results are displayed in a grid that can resize to fit more or less divs per row based on the width of the browser.

C. HTML/CSS & Media | D. Code Conventions

  • HTML and CSS are verified. CSS is contained in an external style sheet.
  • Code is contained in an external JavaScript file. Code uses let and const (no var) and querySelector() and querySelectorAll().
  • Each function has comments explaining what it does. All calls to console.log() are commented out.

Things of Note

  • I am particularly proud of how the tags turned out. I feel that the process of adding and removing custom tags is very smooth and satisfying, and it took me quite a while to get it right. Connecting the "X" buttons to their respective tags was particularly difficult, but I'm very happy with how it turned out.
  • I wsa very particular with this project because I intend to use this tool in the future. FreeSound is a database of user-uploaded sounds and samples licensed under Creative Commons, and it's a really great place to find free-to-use sound effects. I use it all the time for my IGM projects. However, the interface of FreeSound sometimes bothers me, so I created this tool to allow me to access FreeSound and search for sound effects in a more intuitive way. Perhaps if this project makes it onto the list of example projects, future students can make use of it as well?
  • There are is a lot more that I want to add to this project, but given the time crunch I felt this was a good place to end for now. However, I will probably continue to work on this after this submission. In the future I would like to add more search parameters (see brainstorm) and more information displayed with the results. This API is very extensive and the search parameters are incredibly flexible, and I'd like to incorporate some more of that information into the results. For example, displaying ratings, number of downloads, the date uploaded, file types and file size, etc.
